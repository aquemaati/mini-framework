Gestion de l'tat Interne

Vous pouvez grer l'tat interne d'un composant en utilisant this.state et this.setState(newState). Lorsque vous appelez setState, le composant se re-rend automatiquement avec le nouvel tat.

Exemple : Compteur Interne

// src/components/my-counter.ts

import { createComponent } from '../core/createComponent';

createComponent('my-counter', {
  state: { count: 0 }, // tat initial

  render(state) {
    return `
      <div>
        <p>Compteur : ${state.count}</p>
        <button class="increment">Incrmenter</button>
      </div>
    `;
  },

  events: {
    'click@.increment': function () {
      const currentCount = this.state.count;
      this.setState({ count: currentCount + 1 });
    },
  },
});

Explications :

  • tat Initial : Le compteur dmarre 0 grce state: { count: 0 }.
  • Mise Jour de l'tat : Lorsqu'on clique sur le bouton, setState est appel avec le nouveau compteur.
  • Rendu Automatique : Le composant se met jour automatiquement pour reflter le nouvel tat.

Utilisation dans le HTML :

<my-counter></my-counter>

Observation des Attributs

Vous pouvez rendre votre composant ractif aux changements d'attributs en utilisant observedAttributes et attributeChangedCallback.

Exemple : Composant Ractif aux Attributs

// src/components/user-greeting.ts

import { createComponent } from '../core/createComponent';

createComponent('user-greeting', {
  render() {
    const name = this.getAttribute('name') || 'Utilisateur';
    return `<p>Bonjour, ${name} !</p>`;
  },

  observedAttributes: ['name'],

  attributeChangedCallback(name, oldValue, newValue) {
    if (name === 'name') {
      this.update();
    }
  },
});

Utilisation dans le HTML :

<user-greeting name="Alice"></user-greeting>

Modification Dynamique :

<script>
  const greeting = document.querySelector('user-greeting');
  greeting.setAttribute('name', 'Bob');
</script>

Gestion Globale de l'tat avec le Store

Le Store est un objet global qui vous permet de grer l'tat partag entre plusieurs composants.

Accs l'tat Global

Pour accder l'tat global, importez le store et utilisez store.getState().

Exemple :

import { store } from '../core/store';

const currentState = store.getState();

Mise Jour de l'tat Global

Pour mettre jour l'tat global, utilisez store.setState(newState). Cela fusionnera newState avec l'tat existant et notifie tous les abonns.

Exemple :

store.setState({ user: { name: 'Alice' } });

Abonnement aux Changements d'tat

Vous pouvez vous abonner aux changements d'tat en utilisant store.subscribe(listener). Le listener sera appel chaque fois que l'tat est mis jour.

Exemple :

store.subscribe((newState) => {
  console.log('Nouvel tat:', newState);
});

Exemple Complet : Compteur Global

Cration du Composant :

// src/components/global-counter.ts

import { createComponent } from '../core/createComponent';
import { store } from '../core/store';

createComponent('global-counter', {
  render() {
    const count = store.getState().count || 0;
    return `
      <div>
        <p>Compteur Global : ${count}</p>
        <button class="increment">Incrmenter</button>
      </div>
    `;
  },

  events: {
    'click@.increment': function () {
      const currentCount = store.getState().count || 0;
      store.setState({ count: currentCount + 1 });
    },
  },

  connectedCallback() {
    this.unsubscribe = store.subscribe(() => {
      this.update();
    });
  },

  disconnectedCallback() {
    if (this.unsubscribe) {
      this.unsubscribe();
    }
  },
});

Explications :

  • Accs l'tat Global : Le compteur lit la valeur actuelle de count depuis le store.
  • Mise Jour de l'tat : En cliquant sur le bouton, le compteur incrmente la valeur de count dans le store.
  • Abonnement : Le composant s'abonne aux changements du store pour se mettre jour lorsque l'tat change.

Utilisation dans le HTML :

<global-counter></global-counter>

Systme de Routage Intgr

Introduction au Routage

Le systme de routage vous permet de synchroniser l'URL avec l'tat de votre application, permettant une navigation fluide entre diffrentes vues sans rechargement de la page.

Dfinition des Routes

Les routes sont dfinies en associant un chemin un composant.

Exemple : Dfinition des Routes dans main.ts

// src/main.ts

import { router } from './core/router';

// Dfinition des routes
router.addRoute('/', 'home-page');
router.addRoute('/about', 'about-page');
router.addRoute('/contact', 'contact-page');
router.addRoute('/articles', 'article-list');
router.addRoute('/article/:id', 'article-detail');

// Gestion de la route initiale
router.handleRoute();

Navigation entre les Vues

Utilisez le composant <router-link> pour crer des liens qui naviguent entre les vues en modifiant l'URL sans recharger la page.

Exemple :

<nav>
  <router-link href="/">Accueil</router-link>
  <router-link href="/about"> Propos</router-link>
  <router-link href="/contact">Contact</router-link>
</nav>

Composant router-link

Le composant <router-link> encapsule une balise <a> et intercepte les clics pour utiliser le routeur au lieu du comportement par dfaut du navigateur.

Dfinition du Composant :

// src/components/router-link.ts

import { createComponent } from '../core/createComponent';
import { router } from '../core/router';

createComponent('router-link', {
  render() {
    const href = this.getAttribute('href') || '#';
    return `<a href="${href}"><slot></slot></a>`;
  },

  events: {
    'click@a': function (event: Event) {
      event.preventDefault();
      const href = this.getAttribute('href')!;
      router.navigate(href);
    },
  },
});

Gestion des Paramtres de Route

Le routeur supporte les paramtres dynamiques dans les chemins de route en utilisant la syntaxe :param.

Exemple : Route avec Paramtre

// Dans main.ts
router.addRoute('/article/:id', 'article-detail');

Dans le Composant :

// src/components/article-detail.ts

import { createComponent } from '../core/createComponent';
import { router } from '../core/router';

createComponent('article-detail', {
  render() {
    const id = router.getParams().id;
    const article = getArticleById(id);

    if (article) {
      return `
        <h1>${article.title}</h1>
        <p>${article.content}</p>
      `;
    } else {
      return `<p>Article non trouv</p>`;
    }
  },
});

// Fonction simule pour rcuprer un article
function getArticleById(id: string | undefined) {
  const articles = [
    { id: '1', title: 'Article 1', content: 'Contenu de l\'article 1' },
    // ... autres articles
  ];
  return articles.find((article) => article.id === id);
}

Gestion des Routes Non Trouves

Si aucune route ne correspond l'URL actuelle, le routeur affiche le composant not-found.

Exemple :

// src/components/not-found.ts

import { createComponent } from '../core/createComponent';

createComponent('not-found', {
  render() {
    return `
      <h1>Page Non Trouve</h1>
      <p>La page que vous recherchez n'existe pas.</p>
      <router-link href="/">Retour  l'accueil</router-link>
    `;
  },
});
